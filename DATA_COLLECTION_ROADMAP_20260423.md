## 🔴 최우선 (기존 BT 한계 해소)

### 1. 종목별 일간·분간 수급 (외국인 / 기관 / 개인)

**왜 필요한가**
BT-06의 본질. 4/23에 SOX +2.72%인데 SK하이닉스 −0.08%였던 이유는 외국인 −₩332.5B 순매도. 이 수급 데이터가 있어야 "sox_up AND 외국인 순매수" 조건 검증 가능.

**수집 방법**
- **일간**: pykrx `get_market_net_purchases_of_equities_by_ticker` — 즉시 가능
- **분간**: 차트 프로그램 분간 수급 데이터 (있으면 수집, 없어도 무방)

**우선도**: ★★★★★
**용량**: 일간 × 2년 × 150종목 ≈ 10MB

---

### 2. 호가 (10호가 또는 5호가) 스냅샷

**왜 필요한가**
슬리피지 모델링. 현재 백테스트는 0.1% 고정 가정인데 실제 KOSDAQ 중소형은 ±0.3~0.5%. BT-04/05 파라미터가 "지나치게 낙관적"일 가능성.

**수집 방법**
- KIS API `inquire-asking-price`
- 차트 프로그램의 Level 2
- **초당 저장은 비현실적** — "주문 직전 5초 스냅샷" 또는 분봉 close 시점에만 저장

**우선도**: ★★★★★
**용량**: 주문당 수 KB → 일 100 주문 × 6개월 = 600MB 이내

---

### 3. 체결 강도 / 매수·매도 체결 비율

**왜 필요한가**
1분봉 close만으로는 "누가 사고 누가 팔았는지" 모름. 체결강도가 100 이상이면 매수세 우위. 단기 모멘텀 판단에 필수.

**수집 방법**
- KIS API `inquire-price` 반환의 `체결강도`
- 일부 증권사 차트는 분봉에 내재 (컬럼 확인)

**우선도**: ★★★★
**용량**: 1분봉 스키마에 컬럼 추가 → 증분 미미

---

## 🟠 고가치 (신규 BT 가능)

### 4. 섹터 지수 1분봉 (KRX 섹터별)

**왜 필요한가**
"반도체 섹터 강세인가"를 SK하이닉스만으로 판단하는 한계. KRX 반도체 지수, KOSPI IT, KRX 2차전지 등 섹터 지수 분봉으로 섹터 모멘텀 정확 측정.

**수집 방법**
- pykrx `get_index_ohlcv` (섹터지수)
- 차트 프로그램

**우선도**: ★★★★
**용량**: 10~20개 지수 × 분봉 2년 ≈ 200MB

### 6. 공매도 잔고 (일간)

**왜 필요한가**

## 💾 저장 설계 권장

### 포맷 & 디렉터리 구조

```
D:\Project\market_data_warehouse\
├── stocks_1min/              # 종목 1분봉
│   ├── 005930/
│   │   ├── 2026-04.parquet   # 월 단위 파티션
│   │   └── 2026-05.parquet
│   └── 000660/...
├── flows_daily/              # 수급 (일간)
│   └── 2026.parquet
├── indices_1min/             # 섹터 · 선물 지수 1분봉
├── shorts_daily/             # 공매도 잔고
├── options_daily/            # Put/Call ratio
└── metadata/
    └── universe_history.parquet  # 종목 유니버스 이력 (편입/편출)
```

### 포맷 선택 이유

- **Parquet** (pyarrow): 컬럼 압축 + pandas 직접 읽기. 1분봉 1년 ≈ 500MB → 100MB로 축소
- **DuckDB**: SQL 쿼리 필요 시 Parquet 파일을 직접 쿼리 가능 (별도 import 불필요)
- **피해야 할 것**: CSV (용량 폭증), Supabase (분봉 저장은 비용·지연 문제)

### 분봉 데이터 스키마 (최소)

```python
columns = [
    "datetime",        # tz-aware KST
    "open", "high", "low", "close", "volume",
    "value",           # 거래대금 (원)
    "trade_strength",  # 체결강도 (수집 가능하면)
    "bid_vol", "ask_vol",  # 매수·매도 체결량 분리 (가능하면)
]
```

### 수급 데이터 스키마

```python
columns = [
    "date",            # YYYY-MM-DD
    "code",
    "foreign_net",     # 외국인 순매수 (원)
    "institution_net", # 기관 순매수 (원)
    "individual_net",  # 개인 순매수 (원)
    "foreign_holdings_pct",  # 외국인 보유 비중 (%)
]
```
